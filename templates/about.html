Network Telemetry Framework

Our network telemetry framework provides real-time insights into network performance, health, and status across three key areas of the campus.

Monitor critical metrics like bandwidth usage, packet loss, latency, and routing information from various network devices including routers, switches, and firewalls.

  • Real-time data visualization
  • Security threat detection
  • Performance monitoring
  • Device health tracking

Project Overview

Our Network Telemetry Framework aims to develop a robust system for real-time monitoring and management of network performance, supporting network operators in gaining insights into their network's health and status.

Key Features

The framework extracts performance metrics from network devices, transports telemetry data efficiently, and provides interactive visualization for network health status and security threat detection.

Architecture

Our system consists of four planes: Management, Control, Forwarding, and External Data Telemetry, working together to ensure comprehensive network monitoring with minimal overhead.

Technologies

We utilize Mininet for network simulation, Apache Kafka for data streaming, Golang for network communication, and Python for data analysis and machine learning.
